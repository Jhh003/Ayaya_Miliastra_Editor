# 安全策略

## 报告安全问题
如果你发现可能的安全漏洞、滥用风险或隐私泄露问题，请**不要**直接在公开 Issue 中披露细节。

优先使用 GitHub 的 **Private Vulnerability Reporting / Security Advisory**（如果仓库已开启）。
若仓库未开启该功能，请提交一个不含敏感细节的 Issue，并注明“需要私下沟通”，维护者会在后续给出安全沟通渠道。

## 支持范围
- 本仓库仅用于离线教学/模拟与工具链研究。
- 不支持、也不接受任何试图将自动化脚本对接官方《原神》客户端/服务器、规避官方安全策略的需求与提交。

## 版本与修复
展示型仓库的修复会通过 PR 合入；建议在 Issue 中提供：
- 最小复现步骤（避免包含私密资源）
- 错误堆栈/日志（去除敏感信息）
- Windows/Python 版本
