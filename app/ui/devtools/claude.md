## 目录用途
`ui/devtools/` 存放仅面向开发阶段与调试用途的 UI 辅助工具，例如悬停检查器等。这些工具可以依赖具体业务面板与 UI 结构，用于帮助在运行时快速定位控件归属与源码位置，但不参与正式业务流程和正常用户操作路径。

- 典型内容：
  - 基于悬停/点击的控件高亮与信息浮层（例如 `WidgetHoverInspector`）
  - 将 Qt 控件映射回工程源码文件的路径猜测逻辑
  - 与右侧属性面板、配置组件注册表等业务 UI 的轻量集成，方便排查字段来源与配置 ID

## 当前状态
- `view_inspector.py`：提供 `WidgetHoverInspector` 类，通过在应用级安装事件过滤器实现：
  - 鼠标在主窗口内移动时，高亮当前命中的控件矩形；
  - 在主窗口右上角展示包含 Qt 类名、Python 类/模块名、objectName、所属 Panel 标题、所属 Tab 文本、配置组件 ID 等信息的可复制文本；
  - 支持左键点击锁定当前高亮控件，右键点击信息标签复制所有调试信息到剪贴板；
  - 通过 `app.ui.panels.config_component_registry.find_config_component` 解析右侧属性 Panel 中 `QGroupBox` 的配置组件 ID，帮助将 UI 分组与 `engine.configs`/管理配置映射起来。
- 本目录代码默认不在普通运行路径中启用，需要由主窗口等上层通过快捷键或设置显式开启（例如 F12 切换悬停检查器）。

## 注意事项
- 此目录中的模块可以依赖 `app.ui.panels`、`engine.*` 等业务代码，但 **不得反向成为业务流程的强制依赖**：应用在未导入或未启用这些工具时仍需完整可用。
- 开发工具的开关应集中由上层 UI（如 `MainWindowV2`）控制，例如通过快捷键或“开发者模式”开关启用/关闭事件过滤器与叠加层。
- 新增开发工具时，应确保：
  - 不引入新的磁盘读写与网络访问职责，仅做 UI 层调试与信息展示；
  - 不在构造函数中做重型计算或长耗时操作，避免在启用时卡顿；
  - 将所有调试信息封装为易复制的文本形式，方便在 issue 或日志中粘贴使用。
