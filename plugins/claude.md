# 目录用途
可插拔实现层：节点实现与与环境耦合的扩展（如自动化适配）。

# 公共 API
通过“静态注册表”对引擎暴露能力，不直接被 `engine` 依赖。

# 依赖边界
- 允许依赖：`engine/*`
- 禁止依赖：`app/*`（除装配）、`assets/*`（除只读数据）

# 当前状态
- 节点实现已按 `server` / `client` / `shared` 分域存放，静态注册表 `plugins/nodes/registry.py` 随仓库版本生成并与当前节点清单一致。
- 新增或调整节点实现后需运行 `tools/generate_plugin_registry.py` 重新生成注册表（含懒加载路径与全角文件名支持），插件层不做运行时动态扫描。

# 注意事项
- 注册方式采用“静态注册（生成器维护）”：运行时只读静态清单，避免动态扫描。
