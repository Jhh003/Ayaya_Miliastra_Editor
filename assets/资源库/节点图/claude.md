# 节点图资源

## 目录用途
存放节点图的 `.py` 代码文件与子文件夹。

## 说明
- 节点图仅使用 `.py` 保存；不支持 `.json`。
- 允许创建空文件夹，索引会一并扫描。

## 文档约定（重要）
- 这里只描述目录用途与规范，不记录具体资源清单/不解释单个资源是做什么的
- 本 `claude.md` **只描述目录用途、子目录划分与通用约束**。
- **不在此列举本目录下具体有哪些节点图文件**，也不解释单个节点图的业务含义。

## 子目录结构
- `server/`：服务器端节点图，可按子文件夹组织，文件命名与 `graph_id` 语义对齐
- `client/`：客户端节点图（如技能节点图、本地过滤器节点图），同样按子文件夹组织，`folder_path` 建议以用途前缀开头（如 `技能节点图/锻刀英雄`）

## 注意事项与边界条件
- 读写通过 `engine.resources.resource_manager.ResourceManager`；该管理器使用缓存与指纹（fingerprint）机制避免每次全量扫描资源库。手工编辑需保持代码可解析，以便资源索引与指纹计算正常工作。
- 代码文件顶部必须包含元信息注释（graph_id, graph_name, graph_type, folder_path）。
- 节点名称必须与 `node_implementations/` 中实现函数的 `@node_spec` 定义保持一致。
- if 条件必须来自布尔型节点或其输出变量；禁止直接书写任何 Python 比较表达式（如 `A == B`、`X is None`）以及 `if not ...`，取反请使用【逻辑非】。
- 长连线优化：当变量跨越较多流程节点被复用时，建议在使用点前通过【获取局部变量(初始值=变量)】进行就近中继，然后从其“值”输出继续连线。
- 命令行验证入口（逐行执行，PowerShell 不使用 `&&`）：`python -X utf8 -m tools.validate.validate_graphs --all`

### 校验入口
- 使用项目统一的节点图/复合节点校验入口进行校验。
- 如需仅校验资源库内节点图，可使用资源库配套的快速校验入口（不在本文档写死具体脚本文件名）。
