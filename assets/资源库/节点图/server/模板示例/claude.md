## 目录用途
- 存放服务器端节点图的教学示例脚本（类结构 Python）。
- 示例聚焦“如何写”，用于演示信号、复合节点、延迟执行、循环与局部变量等典型写法，便于在编辑器中对照学习。

## 文档约定（重要）
- 这里只描述目录用途与规范，不记录具体资源清单/不解释单个资源是做什么的
- 本 `claude.md` **只描述目录用途与编写/提交规范**。
- **不在此列举具体示例脚本文件名**，也不解释单个示例的业务含义。

## 当前状态
稳定可用。示例覆盖常见节点图写法与静态解析约束（不在本文档维护清单）。

## 注意事项
- 脚本需保持与引擎节点图解析约定一致：文件头 docstring 中声明 `graph_id/graph_name/graph_type/description`，类名与 `graph_name` 对齐，`GRAPH_VARIABLES` 中变量名必须与图中通过【设置节点图变量】访问的变量名一致。
- 示例力求精简，仅保留最小可运行路径；必要说明写在类/方法 docstring 中，便于 UI 中直接阅读。
- 分支写法优先使用“正向条件 + else 提前返回/执行”，避免出现 `if 逻辑非运算(...)` 导致“主流程接否分支”的阅读负担。
- 对“是否相等”等比较类节点，关键字参数应使用节点端口名（例如 `输入1` / `输入2`），避免端口名不匹配导致校验失败。
- 节点图脚本必须保持为“可被 Python AST 解析的合法代码”（包括缩进），否则资源索引扫描时会因解析失败而阻断启动。
- 如需保留调试镜像类图变量，建议使用 `调试_` 前缀并把语义写全，避免与业务变量混淆。
- 本目录仅存放手写教学示例源码（`.py`）；不提交工具生成的临时导出文件。
